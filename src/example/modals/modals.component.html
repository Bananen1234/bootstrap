Modals
Javascript needed! Please note that the examples below is just a representation of how modals look and the needed markup. In order to use modals you need javascript to control when and how to show them.

Using Angular? A simple and elegant solution to add modals to an angular app and at the same time allow browser history to determine modal state is to leverage the angular router and add the modal markup to route component. If you don't want to use routes you could also take a look at ng-bootstrap and their modal component.

Mobile development! To make the experience extra crisp on mobile devices and more specifically to avoid the issue with scrolling content behind the modal, you should consider adding body scroll lock to your project when implementing modals. An example implementation for angular can also be found on stack blitz.

Add .modal-lg class to the modal dialog to get a larger modal, works with normal modals and aside modals.

Add the class .modal-dialog-scrollable to the modal element to allow the body content to overflow i.e. scroll. If you want the buttons or modal actions to align nicely at the bottom put them inside a modal footer with the class .border-top to get a border against the scrolling content body. This goes for aside and fullscreen modals too!

Add .modal-aside class to the modal element together with either .modal-aside-right or .modal-aside-left to get a modal that will slide in from the side.

Add .modal-fullscreen class to the modal element to get a fullscreen modal.
